Имеется класс с несколькими состояниями (GumballMachine), он может менять эти состояния runtime,
он делегирует реализацию конкретному состоянию. Это позволяет избежать множественных конструкций if,
соблюдать принцип открытости для расширения и закрытости для изменения, локализовать поведение
каждого состояния в отдельном классе.
Если переходы состояния статичны, их лучше осуществлять в классе-контексте (GumballMachine), если
динамичны (например, переход зависит от кол-ва мячей), то в классах состояний.
