Паттерн применяется при существовании некторого количества сущностей, выполняющие одни действия, но
каждая по-своему. Дополнительная гибкость - если заранее не известно количество реализаций, их можно
дополнять, без переписывания имеющегося кода. Имеется возможность изменить реализацию поведения во
время исполнения программы.
Имеются: базовый абстрактный класс, содержащий поле интерфейса и метод set(). Классы-наследники
абстрактного класса. Отдельные классы с реализацией интерфейса.
